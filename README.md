# Payment-middle-platform

系统定位：
该系统用于微信生态下的 H5 支付场景，面向多个业务平台统一接入支付能力。用户通过微信扫码打开 H5 页面完成付款，后台用于管理业务平台、支付账号、支付路由、支付订单和基础退款。

核心业务关系：
1. 后台账号与业务平台是一对一关系：后台账号 1 : 1 业务平台。
2. 业务平台与支付账号是一对多关系：业务平台 1 : N 支付账号。
3. 用户登录后不需要选择平台，系统根据当前登录账号关联的业务平台自动进入后台。
4. 进入后台后，所有订单、支付账号、路由规则、退款数据都只展示当前账号所属平台的数据。

后台整体布局：
1. 左侧固定导航栏。
2. 顶部状态栏。
3. 主内容区。
4. 列表页包含筛选区、表格区、分页。
5. 新增 / 编辑操作优先使用右侧抽屉或弹窗。
6. 详情信息可以使用详情页、抽屉或弹窗展示。

左侧导航菜单：
1. 首页看板
2. 账号与平台关联
3. 业务平台管理
4. 支付账号管理
5. 支付路由配置
6. 支付订单管理
7. 退款管理

顶部栏内容：
1. 当前平台名称
2. 当前登录用户
3. 退出登录

视觉风格：
1. 专业、简洁、稳重，适合支付和财务类后台。
2. 以白色、浅灰、深蓝、绿色作为主色调。
3. 页面信息密度适中，方便运营和财务人员长期使用。
4. 按钮、筛选器、表格、状态标签要清晰可识别。
5. 启用、停用、异常、成功、失败等状态要使用不同颜色标签。
6. 支付金额要突出展示，金额单位为元。
7. 不要生成官网首页、营销落地页、炫酷大屏或移动端页面。
8. 请生成桌面端 Web 后台原型，风格参考 Ant Design Pro、Element Plus Admin、腾讯云控制台、微信支付商户平台后台。
